# 3.2 Visual Regression Testing

## Goal
Automate screenshot comparison to prevent silent UI breaks during refactors.

---

## Errors to Find

| # | Error | Difficulty |
|---|-------|------------|
| 3.2.1 | No visual regression tests configured | Medium |
| 3.2.2 | Critical components without snapshot coverage | Medium |

---

## Components to Cover

- Hero section (above the fold)
- Modal (open state)
- Tabs (each state)
- FAQ accordion (expanded/collapsed)

---

## Playwright Setup

```js
// tests/visual.spec.js
import { test, expect } from '@playwright/test';

test('hero section', async ({ page }) => {
  await page.goto('/');
  await expect(page.locator('.hero')).toHaveScreenshot();
});

test('modal open', async ({ page }) => {
  await page.goto('/');
  await page.click('[data-open-modal]');
  await expect(page.locator('.modal')).toHaveScreenshot();
});
```

---

## Validation

1. Run `npx playwright test`
2. Review generated snapshots
3. Verify critical states are covered
