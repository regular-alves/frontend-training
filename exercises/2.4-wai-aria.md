# 2.4 WAI-ARIA Patterns

## Goal
Implement accessible states for dynamic components.

---

## Errors to Find

| # | Error | Difficulty |
|---|-------|------------|
| 2.4.1 | Tabs without `role="tablist"`, `role="tab"`, `aria-selected` | Hard |
| 2.4.2 | FAQ accordion without `aria-expanded`, `aria-controls` | Hard |
| 2.4.3 | Modal without `role="dialog"`, `aria-modal`, focus trap | Hard |

---

## What to Look For

### Tabs
```html
<!-- Missing ARIA -->
<div class="tabs">
  <div class="tab active">Day 1</div>
</div>

<!-- Should have -->
<div role="tablist">
  <button role="tab" aria-selected="true">Day 1</button>
</div>
```

### Accordion
```html
<!-- Missing ARIA -->
<div class="faq-question" onclick="toggle()">

<!-- Should have -->
<button aria-expanded="false" aria-controls="answer-1">
```

### Modal
```html
<!-- Missing ARIA -->
<div class="modal">

<!-- Should have -->
<div role="dialog" aria-modal="true" aria-labelledby="modal-title">
```

---

## Validation

1. Test with screen reader (VoiceOver, NVDA)
2. Verify keyboard navigation (Tab, Enter, Esc)
3. Check focus management (focus trap in modal)
